## 环境差异对比

### 1. 日志配置

| 环境 | 根日志级别 | 业务日志级别 | SQL 日志 | 日志格式 | 脱敏 |
|------|-----------|-------------|---------|---------|------|
| dev | INFO | DEBUG | DEBUG | 可读格式 | ❌ |
| test | INFO | DEBUG | INFO | JSON | ✅ |
| prod | INFO | INFO | INFO | JSON | ✅ |

### 2. 链路追踪采样率

| 环境 | 采样率 | 说明 |
|------|--------|------|
| dev | 100% | 开发环境全量采样，方便调试 |
| test | 50% | 测试环境半量采样 |
| prod | 10% | 生产环境低采样率，降低性能开销 |

### 3. 数据库配置

| 环境 | 数据库名 | Redis DB | MongoDB DB |
|------|---------|----------|-----------|
| dev | openiot_dev | 0 | openiot_dev |
| test | openiot_test | 1 | openiot_test |
| prod | openiot (环境变量) | 2 | openiot (环境变量) |

### 4. 线程池配置

#### Connect Service (Netty)

| 环境 | Boss Threads | Worker Threads | Backlog | 缓冲区大小 |
|------|-------------|----------------|---------|-----------|
| dev | 1 | 0 (自动) | 2048 | 65535 |
| test | 1 | 0 (自动) | 4096 | 65535 |
| prod | 2 | 0 (自动) | 8192 | 131072 |

#### Data Service (Kafka Consumer)

| 环境 | Concurrency | Max Poll Records | Fetch Min Size |
|------|------------|------------------|----------------|
| dev | 2 | 100 | 1024 |
| test | 3 | 300 | 1024 |
| prod | 5 | 500 | 2048 |

## 如何使用

### 方式 1: IDEA 启动（推荐开发环境）

1. 打开 IDEA 的 **Run/Debug Configurations**
2. 在 **Active profiles** 中输入: `dev`
3. 启动服务

### 方式 2: 命令行启动

```bash
# 开发环境
java -jar device-service.jar --spring.profiles.active=dev

# 测试环境
java -jar device-service.jar --spring.profiles.active=test

# 生产环境
java -jar device-service.jar --spring.profiles.active=prod
```

### 方式 3: 环境变量

```bash
# Windows PowerShell
$env:SPRING_PROFILES_ACTIVE="prod"

# Linux/Mac
export SPRING_PROFILES_ACTIVE=prod
```

### 方式 4: Docker 启动

```bash
# 开发环境
docker run -e SPRING_PROFILES_ACTIVE=dev device-service:latest

# 生产环境
docker run \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e DB_HOST=prod-db.internal \
  -e DB_USERNAME=openiot \
  -e DB_PASSWORD=secret \
  -e REDIS_HOST=prod-redis.internal \
  -e REDIS_PASSWORD=secret \
  device-service:latest
```

## 生产环境必需的环境变量

### 所有服务通用

```bash
# 数据库
DB_HOST=prod-db.internal
DB_NAME=openiot
DB_USERNAME=openiot
DB_PASSWORD=your_password

# Redis
REDIS_HOST=prod-redis.internal
REDIS_PASSWORD=your_password

# 链路追踪
OTEL_HOST=prod-otel-collector
```

### Connect Service 额外变量

```bash
# Kafka
KAFKA_SERVERS=prod-kafka1:9092,prod-kafka2:9092,prod-kafka3:9092
```

### Data Service 额外变量

```bash
# MongoDB
MONGODB_USERNAME=openiot
MONGODB_PASSWORD=your_password
MONGODB_HOST=prod-mongo1,prod-mongo2,prod-mongo3
```

## 配置优先级

Spring Boot 配置的优先级（从高到低）:

1. 命令行参数 `--spring.profiles.active=prod`
2. 环境变量 `SPRING_PROFILES_ACTIVE`
3. `application-{profile}.yml` (如 application-prod.yml)
4. `application.yml` (公共配置)

## 动态调整日志级别（无需重启）

使用 Spring Boot Actuator 动态调整日志级别:

```bash
# 查看当前日志级别
curl http://localhost:8081/actuator/loggers/com.openiot

# 动态修改日志级别
curl -X POST http://localhost:8081/actuator/loggers/com.openiot \
  -H "Content-Type: application/json" \
  -d '{"configuredLevel": "DEBUG"}'
```

## 注意事项

### 1. 开发环境

- ✅ 使用可读日志格式，方便调试
- ✅ SQL 日志 DEBUG 级别，可以看到执行的 SQL
- ✅ 100% 链路追踪采样
- ❌ 不开启敏感数据脱敏

### 2. 测试环境

- ✅ 使用 JSON 日志格式，与生产环境一致
- ✅ 50% 链路追踪采样
- ✅ 开启敏感数据脱敏
- ❌ 禁止 Flyway 清理数据库

### 3. 生产环境

- ✅ 使用 JSON 日志格式，便于 ELK 采集
- ✅ 10% 链路追踪采样，降低性能开销
- ✅ 必须开启敏感数据脱敏
- ✅ 使用环境变量管理敏感配置
- ✅ 禁止 Flyway 清理数据库
- ✅ 优化线程池和连接池配置

## 问题排查

### Q1: 日志没有输出到控制台?

**原因**: Logback 配置使用了错误的 Appender

**解决**: 检查 `logback-spring.xml` 中开发环境是否使用 `ASYNC_PRETTY` appender

### Q2: 端口号没有显示?

**原因**: 日志级别设置过高，或使用了 JSON 格式

**解决**: 开发环境使用 `ASYNC_PRETTY` appender

### Q3: Profile 没有生效?

**检查方式**:
1. 查看启动日志中的 `The following 1 profile is active: "xxx"`
2. 确认 `application-{profile}.yml` 文件存在
3. 确认 `spring.config.activate.on-profile` 配置正确

### Q4: 生产环境配置无法读取环境变量?

**检查方式**:
1. 确认环境变量名称正确（如 `DB_HOST`）
2. 确认使用 `${DB_HOST:default_value}` 语法提供默认值
3. 在容器环境中确认环境变量已正确注入
