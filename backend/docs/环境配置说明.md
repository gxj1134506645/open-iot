# Spring Boot 环境配置说明

## 配置文件结构

```
src/main/resources/
├── application.yml           # 公共配置（所有环境共享）
├── application-dev.yml       # 开发环境配置
├── application-test.yml      # 测试环境配置
└── application-prod.yml      # 生产环境配置
```

## Profile 激活方式

### 方式 1: IDEA 启动配置（推荐）

1. 打开 Run/Debug Configurations
2. 在 **Active profiles** 中输入: `dev`
3. 启动服务

### 方式 2: 命令行参数

```bash
# 开发环境
java -jar device-service.jar --spring.profiles.active=dev

# 测试环境
java -jar device-service.jar --spring.profiles.active=test

# 生产环境
java -jar device-service.jar --spring.profiles.active=prod
```

### 方式 3: 环境变量

```bash
# Linux/Mac
export SPRING_PROFILES_ACTIVE=prod

# Windows PowerShell
$env:SPRING_PROFILES_ACTIVE="prod"

# Docker
docker run -e SPRING_PROFILES_ACTIVE=prod device-service:latest
```

## 各环境配置差异

### 开发环境 (dev)

**特点**:
- 日志级别: DEBUG（详细日志）
- 日志格式: 可读格式（ASYNC_PRETTY）
- 链路追踪采样: 100%
- 敏感数据脱敏: 关闭
- 数据库: `openiot_dev`
- Redis Database: 0

**日志输出示例**:
```
2026-03-03 10:15:30.123 [main] [N/A] [N/A] DEBUG c.o.d.service.DeviceService - 查询设备列表...
```

### 测试环境 (test)

**特点**:
- 日志级别: DEBUG
- 日志格式: JSON 格式（ASYNC_JSON）
- 链路追踪采样: 50%
- 敏感数据脱敏: 开启
- 数据库: `openiot_test`
- Redis Database: 1

**日志输出示例**:
```json
{
  "@timestamp": "2026-03-03T10:15:30.123+0800",
  "level": "DEBUG",
  "message": "查询设备列表...",
  "service": "device-service",
  "env": "test",
  "traceId": "abc123",
  "tenantId": "tenant001"
}
```

### 生产环境 (prod)

**特点**:
- 日志级别: INFO（只记录重要日志）
- 日志格式: JSON 格式（ASYNC_JSON）
- 链路追踪采样: 10%（降低性能开销）
- 敏感数据脱敏: 开启
- 数据库: 从环境变量读取
- Redis Database: 2

**日志输出示例**:
```json
{
  "@timestamp": "2026-03-03T10:15:30.123+0800",
  "level": "INFO",
  "message": "设备创建成功",
  "service": "device-service",
  "env": "prod",
  "traceId": "xyz789",
  "tenantId": "tenant001"
}
```

## Logback 配置说明

`logback-spring.xml` 根据 Profile 自动切换日志级别和格式:

```xml
<!-- 开发环境 -->
<springProfile name="dev,local">
    <root level="DEBUG">
        <appender-ref ref="ASYNC_PRETTY"/>  <!-- 可读格式 -->
    </root>
    <logger name="com.openiot" level="DEBUG"/>
</springProfile>

<!-- 测试环境 -->
<springProfile name="test">
    <root level="DEBUG">
        <appender-ref ref="ASYNC_JSON"/>  <!-- JSON 格式 -->
    </root>
    <logger name="com.openiot" level="DEBUG"/>
</springProfile>

<!-- 生产环境 -->
<springProfile name="prod,production">
    <root level="INFO">
        <appender-ref ref="ASYNC_JSON"/>  <!-- JSON 格式 -->
    </root>
    <logger name="com.openiot" level="INFO"/>
</springProfile>
```

## 日志级别优先级

1. **logback-spring.xml** 中的 `<logger>` 配置（最高优先级）
2. **application-{profile}.yml** 中的 `logging.level.*`
3. **application.yml** 中的 `logging.level.*`

## 常见问题

### Q1: 如何在 IDEA 中查看当前激活的 Profile?

在启动日志中查找:
```
The following 1 profile is active: "dev"
```

### Q2: 如何动态调整日志级别（无需重启）?

使用 Spring Boot Actuator:

```bash
# 查看当前日志级别
curl http://localhost:8081/actuator/loggers/com.openiot

# 动态修改日志级别
curl -X POST http://localhost:8081/actuator/loggers/com.openiot \
  -H "Content-Type: application/json" \
  -d '{"configuredLevel": "DEBUG"}'
```

### Q3: 生产环境为什么使用 INFO 级别?

- **性能考虑**: DEBUG 日志量大,影响性能
- **存储成本**: 减少日志存储空间
- **可读性**: 只关注重要信息
- **链路追踪**: 通过 traceId 可以定位问题

### Q4: 开发环境为什么使用可读格式?

- **易读性**: 方便开发者快速定位问题
- **IDE 集成**: IDEA 可以识别日志格式
- **调试效率**: 直接在控制台查看日志

## 最佳实践

1. **开发环境**: 使用 `dev` profile,日志级别 DEBUG
2. **本地调试**: 使用 `local` profile (等同于 dev)
3. **测试环境**: 使用 `test` profile,日志级别 DEBUG
4. **生产环境**: 使用 `prod` profile,日志级别 INFO
5. **敏感信息**: 生产环境必须开启脱敏
6. **采样率**: 生产环境降低链路追踪采样率 (10%)
